The problem

Every app hides mute
in a different place

Zoom puts it bottom-left. Meet hides it behind a toolbar. Teams moves it every update. When you’re screen-sharing — running a demo, walking through code — fumbling for the mute button is awkward and visible to everyone watching.

KeyMute gives you one shortcut that works everywhere. It mutes at the operating-system level, so every app immediately sees silence. No per-app integration. No Accessibility permission. Just ⌘⇧0.

Why it’s different

Built like a tool
you’d write yourself

Most menu bar apps ship a SwiftUI settings window, an NSHostingView, and an entire view-diffing engine — sitting in memory for a screen you open once a month. That’s 50–150 MB of idle RAM for a mute button.

KeyMute has zero windows. The entire UI is an NSStatusItem, an NSMenu, and a single NSPanel for the toast — allocated once, reused forever. No view hierarchies. No layout solver idling in the background.

What you get

Deliberately minimal

⌨

Global shortcut

Default ⌘⇧0, fully customizable. Works from any app, any fullscreen state. Carbon hotkeys — no Accessibility permission.

◼

System-wide mute

Writes to CoreAudio’s device-level mute. Zoom, Meet, Teams, Chrome — they all see silence instantly. Not a per-app hack.

◉

Menu bar status

Always-visible icon flips between mic and mic-slash. Three styles: Adaptive, Colorful, Monochrome. Optional “MUTED” label.

▢

Invisible toast HUD

Shows “Muted” / “Unmuted” briefly. Uses sharingType=none — your audience never sees it during screen share.

⇄

Device picker

Pin a specific mic or follow system default. Per-device mute memory — re-plugging a USB mic restores its last state.

↻

Stays in sync

Listens to CoreAudio property changes. Muting from System Settings or swapping your mic updates KeyMute in real time.

☼

Auto-mute on Focus

Automatically mutes your mic when macOS Focus / Do Not Disturb turns on. Unmutes when it turns off.

☾

Auto-mute on sleep

Close the lid, mic mutes. No hot-mic surprises after you walk away from your desk mid-call.

⚒

Force Mute / Unmute

Separate shortcuts for scripting, Stream Deck, or automation. Independent from the toggle shortcut.

⚡

~700 KB. No Electron.

Native Swift binary. One SPM dependency. Launches at login via SMAppService. Runs forever, costs nothing.

Under the hood

CoreAudio does the real work

how-it-works.txt
You press ⌘⇧0
│
▼
KeyMute calls AudioObjectSetPropertyData
on the default input device
│
▼
┌─────────────────────────────────────┐
│ kAudioDevicePropertyMute = 1 │
│ scope: kAudioDevicePropertyScopeInput │
└─────────────────────────────────────┘
│
├──▶ Zoom → sees silence
├──▶ Meet → sees silence
├──▶ Teams → sees silence
└──▶ Chrome → sees silence

Setting kAudioDevicePropertyMute on the default input device is equivalent to flipping the mute switch in System Settings → Sound → Input. Because it’s at the device level, every application reading from the mic immediately gets silence — including apps that show their own mute UI.

Install in one line

One-liner (recommended)
curl -fsSL https://raw.githubusercontent.com/ankitaniket/KeyMute/main/install.sh | bash

Downloads the latest release, strips quarantine, verifies the signature, and launches.

Manual (DMG)
  1. Download the latest .dmg from GitHub Releases
  2. Open it → drag Mutify.app → Applications
  3. Right-click → Open → click Open in the Gatekeeper dialog
  4. Press ⌘⇧0 — allow Microphone when prompted
